## Context

PR #19 added a constructor-time preflight to the aider runner that calls `spawnSync('<bin>', ['--version'])` and throws with an install command if ENOENT. The same pattern should apply to the codex-cli runner.

## Proposed change

In `packages/conduit-runner-codex-cli/src/index.ts`, add a `preflight()` method:

1. Take the first whitespace-delimited token of `this.command` (default `codex`, or `codex` from `codex exec ...` if #20 is fixed first).
2. `spawnSync(bin, ['--version'], { stdio: 'ignore' })`.
3. If `result.error.code === 'ENOENT'`, throw with:

   ```
   codex-cli runner: '<bin>' was not found on PATH.

   Install Codex CLI:
     npm install -g @openai/codex

   See https://github.com/openai/codex for setup.
   ```

4. Call `preflight()` from the constructor.

## Order of operations

Recommended to land #20 (codex-cli default-command fix) before this — preflight on the wrong default command wouldn't help much. If both ship together, even better.
